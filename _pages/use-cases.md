<ol class="use-case-list">
    <li>
        <h2 id="1">Improving recruitment</h2>
        <ol>
            <li id="1_1">A data standard to describe a volunteering opportunity</li>
            <li id="1_2">A data standard to describe a volunteer</li>
            <li id="1_3">A standard to match volunteers to opportunities (brokerage)</li>
        </ol>
    </li>
    <li>
        <h2 id="2">Incentivising retention</h2>
        <ol>
            <li id="2_1">A data standard to describe a volunteer’s current and previous work in an organisation or across organisations</li>
        </ol>
    </li>
    <li>
        <h2 id="3">Increasing awareness and marketing</h2>
        <ol>
            <li id="3_1">A data standard to describe a volunteering opportunity and the personal or social outcomes it has, portable between advertising systems</li>
        </ol>
    </li>
    <li>
        <h2 id="4">Making volunteer skill checks, background checks, and training more efficient</h2>
        <ol>
            <li id="4_1">A portable data standard to describe a volunteer’s  current and previous work, training, and accreditations from previous organisations</li>
        </ol>
    </li>
    <li>
        <h2 id="5">Mobilising volunteers in national emergencies (and events)</h2>
        <ol>
            <li id="5_1">A data standard for volunteers and opportunities that is interoperable between national-scale calls for volunteering and local-level recruitment and management</li>
        </ol>
    </li>
    <li>
        <h2 id="6">Building evidence and reports for use in management decisions</h2>
        <ol>
            <li id="6_1">A data standard to describe a volunteer’s work in an organisation that is aggregable and suited to management needs</li>
            <li id="6_2">A means to interpret aggregate data that suits the skills and time constraints of the sector</li>
        </ol>
    </li>
    <li>
        <h2 id="7">Reporting impact (and potential impact) to donors (and potential donors)</h2>
        <ol>
            <li id="7_1">A data standard to describe an organisation’s work and the social benefit it provides.</li>
            <li id="7_2">Controlled, standardised procurement requirements for funders to use.</li>
        </ol>
    </li>
    <li>
        <h2 id="8">Ensuring digital platforms all speak the same language, allowing information to move between systems</h2>
        <ol>
            <li id="8_1">Ensuring there is consensus amongst digital tool providers on a data standard and widespread adoption afterward.</li>
        </ol>
    </li>
</ol>
